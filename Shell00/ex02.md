# Shell00 — Exercise 02 : Oh yeah, mooore...

## Énoncé

**Turn-in directory :** `ex02/`  
**Files to turn in :** `exo2.tar`  
**Allowed functions :** None

Crée les fichiers et dossiers nécessaires pour que `ls -l` affiche **exactement** ceci :

```
total XX
drwx--xr-x  2  XX  XX  XX  Jun  1  20:47  test0
-rwx--xr--  1  XX  XX   4  Jun  1  21:46  test1
dr-x---r--  2  XX  XX  XX  Jun  1  22:45  test2
-r-----r--  2  XX  XX   1  Jun  1  23:44  test3
-rw-r----x  1  XX  XX   2  Jun  1  23:43  test4
-r-----r--  2  XX  XX   1  Jun  1  23:44  test5
lrwxrwxrwx  1  XX  XX   5  Jun  1  22:20  test6 -> test0
```

Une fois fait, archive tout avec :

```sh
tar -cf exo2.tar *
```

---

## Analyse des fichiers à créer

| Fichier | Type | Permissions | Détails |
|---------|------|-------------|---------|
| `test0` | dossier | `drwx--xr-x` | dossier normal |
| `test1` | fichier | `-rwx--xr--` | contient `"Hi "` (4 octets) |
| `test2` | dossier | `dr-x---r--` | dossier normal |
| `test3` | fichier | `-r-----r--` | contient une ligne vide (1 octet) |
| `test4` | fichier | `-rw-r----x` | contient `"A"` (2 octets) |
| `test5` | hard link | `-r-----r--` | lien physique vers `test3` |
| `test6` | lien sym. | `lrwxrwxrwx` | lien symbolique vers `test0` |

## Solution complète

### test0 — dossier `drwx--xr-x`

```sh
mkdir test0
chmod 715 test0
touch -amt 202306012047.00 test0
```

### test1 — fichier `-rwx--xr--`

```sh
echo "Hi " > test1
chmod 714 test1
touch -amt 202306012146.00 test1
```

### test2 — dossier `dr-x---r--`

```sh
mkdir test2
chmod 504 test2
touch -amt 202306012245.00 test2
```

### test3 — fichier `-r-----r--`

```sh
echo > test3
chmod 404 test3
touch -amt 202306012344.00 test3
```

### test4 — fichier `-rw-r----x`

```sh
echo A > test4
chmod 641 test4
touch -amt 202306012343.00 test4
```

### test5 — hard link vers test3

```sh
ln test3 test5
```

> `test5` hérite automatiquement des permissions et du contenu de `test3`.

### test6 — lien symbolique vers test0

```sh
ln -s test0 test6
touch -hamt 202306012220.00 test6
```

> Le `-h` dans `touch` est important : il modifie la date du lien lui-même, pas de la cible.

| `test0` | `drwx--xr-x` | rwx=7 | --x=1 | r-x=5 | **715** |

## Script complet

```sh
#!/bin/sh

# test0 — dossier
mkdir test0
chmod 715 test0
touch -amt 202306012047.00 test0

# test1 — fichier avec contenu "Hi "
echo "Hi " > test1
chmod 714 test1
touch -amt 202306012146.00 test1

# test2 — dossier
mkdir test2
chmod 504 test2
touch -amt 202306012245.00 test2

# test3 — fichier ligne vide
echo > test3
chmod 404 test3
touch -amt 202306012344.00 test3

# test4 — fichier avec "A"
echo A > test4
chmod 641 test4
touch -amt 202306012343.00 test4

# test5 — hard link vers test3
ln test3 test5

# test6 — lien symbolique vers test0
ln -s test0 test6
touch -hamt 202306012220.00 test6

# Vérification
ls -l

# Créer l'archive à rendre
tar -cf exo2.tar *
```

---

## Points importants

- `touch -amt` modifie la date d'accès et de modification
- `-h` dans `touch -hamt` cible le lien symbolique lui-même
- `ln` (sans `-s`) crée un **hard link** — `test5` et `test3` partagent le même inode
- `ln -s` crée un **lien symbolique** — `test6` pointe vers `test0`
- Les permissions du lien symbolique (`lrwxrwxrwx`) sont automatiques, inutile de les `chmod`

# Shell00 - ex00 : Z

## Énoncé

| | |
|---|---|
| **Turn-in directory** | `ex00/` |
| **Files to turn in** | `z` |
| **Allowed files** | `z` |
| **Forbidden functions** | None |

Create a file named `z`.

This file should display the following:

```
?>cat z
Z
?>
```

---

## Solutions possibles

Il existe plusieurs façons de créer ce fichier. Le but est toujours le même : un fichier `z` qui contient exactement `Z` suivi d'un retour à la ligne (`\n`).

### 1. `touch` + éditeur de texte

```sh
touch z
nano z      # ou vim z, vi z
```
Dans l'éditeur, taper `Z` puis valider la ligne (Enter), sauvegarder et quitter.

### 2. `echo`

```sh
echo "Z" > z
```

### 3. `printf`

```sh
printf "Z\n" > z
```

### 4. Heredoc

```sh
cat > z << Hanane
Z
Hanane
```

---

## Comparatif des méthodes

| # | Commande | Ajoute `\n` automatiquement ? | Remarque |
|---|----------|-------------------------------|----------|
| 1 | `touch` + `nano`/`vim`/`vi` | oui | L'éditeur ajoute le `\n` quand la ligne est validée |
| 2 | `echo "Z" > z` | oui | `echo` ajoute toujours `\n`, sauf avec l'option `-n` |
| 3 | `printf "Z\n" > z` | seulement si écrit explicitement | `printf` n'ajoute rien automatiquement, contrairement à `echo` |
| 4 | `cat > z << EOF` ... `EOF` | oui | Chaque ligne tapée dans le heredoc inclut son `\n` |

### Erreurs fréquentes à éviter

```sh
cat z > z           # cat essaie de LIRE z avant qu'il existe -> erreur
printf "Z" > z       # pas de \n -> cat -e z affichera "Z" sans "$"
printf z > Z          # erreur de casse : écrit dans Z (majuscule) et sans \n
```

---

## Vérifier son résultat

```sh
cat -e z
```

- `Z$` affiché -> le `\n` est présent (correct)
- `Z` affiché sans `$` -> il manque le `\n` (incorrect)

```sh
wc -c z
```
Doit afficher exactement `2` (1 caractère `Z` + 1 caractère `\n`).

# Permissions Linux — Comprendre `chmod`

## 1. C'est quoi une permission ?

Chaque fichier Linux a **3 groupes d'utilisateurs** :

```
-r--r-xr-x
│ ─── ─── ───
│  │   │   └── others  (tout le monde)
│  │   └─────  group   (ton groupe)
│  └─────────  owner   (toi)
└────────────  type de fichier (- = fichier, d = dossier, l = lien)
```

Et pour chaque groupe, **3 droits possibles** :

| Lettre | Signification | Sur fichier | Sur dossier |
|--------|---------------|-------------|-------------|
| `r` | read | lire le contenu | voir la liste des fichiers |
| `w` | write | modifier/supprimer | créer/supprimer dedans |
| `x` | execute | lancer comme programme | entrer dans le dossier (`cd`) |
| `-` | aucun droit | — | — |

---

## 2. Lire `-r--r-xr-x` caractère par caractère

```
-  r--  r-x  r-x
│   │    │    │
│   │    │    └── others : r=oui, w=non, x=oui
│   │    └─────── group  : r=oui, w=non, x=oui
│   └──────────── owner  : r=oui, w=non, x=non
└──────────────── fichier normal
```

---

## 3. Les deux formes de `chmod`

### Forme symbolique (lettres)

```sh
chmod u=r,g=rx,o=rx fichier
```

**Qui ?**

| Symbole | Qui ? |
|---------|-------|
| `u` | user (owner) |
| `g` | group |
| `o` | others |
| `a` | all (u+g+o) |

**Opérateur**

| Opérateur | Action |
|-----------|--------|
| `=` | définir exactement |
| `+` | ajouter un droit |
| `-` | enlever un droit |

**Exemples :**

```sh
chmod u+x fichier        # ajoute x au owner seulement
chmod a-w fichier        # enlève w à tout le monde
chmod u=rw,go=r fichier  # owner: rw, group+others: r
```

---

### Forme octale (chiffres)

Chaque droit a une **valeur binaire** :

```
r = 4   (100 en binaire)
w = 2   (010 en binaire)
x = 1   (001 en binaire)
- = 0
```

Tu **additionnes** les valeurs pour chaque groupe :

```
r-- = 4+0+0 = 4
r-x = 4+0+1 = 5
rwx = 4+2+1 = 7
rw- = 4+2+0 = 6
```

Donc pour `-r--r-xr-x` :

```
owner  r-- = 4
group  r-x = 5
others r-x = 5
           ───
           455
```

```sh
chmod 455 fichier
```

---

## 4. Tableau des combinaisons possibles

| Octal | Binaire | Lettres | Signification |
|-------|---------|---------|---------------|
| 0 | 000 | `---` | aucun droit |
| 1 | 001 | `--x` | execute only |
| 2 | 010 | `-w-` | write only |
| 3 | 011 | `-wx` | write + execute |
| 4 | 100 | `r--` | read only |
| 5 | 101 | `r-x` | read + execute |
| 6 | 110 | `rw-` | read + write |
| 7 | 111 | `rwx` | tous les droits |

---

## 5. Pourquoi octal et pas décimal ?

Parce que chaque droit = **1 bit**. 3 droits = **3 bits**. 3 bits max = `111` en binaire = `7` en octal.

```
r w x
1 1 1  →  7
1 0 1  →  5
1 0 0  →  4
```

C'est du **binaire groupé par 3** — l'octal est la représentation naturelle des permissions Unix.  
Ce n'est pas un choix arbitraire, c'est des maths.

## 6. Exercice pratique — ex01

Les permissions cibles : `-r--r-xr-x` = **455**

Construis la commande pour créer `testShell00` avec les bonnes permissions :

```sh
# Étape 1 : créer le fichier
touch testShell00

# Étape 2 : appliquer les permissions
chmod 455 testShell00

# Vérification
ls -l testShell00
# → -r--r-xr-x  1  user  group  ...  testShell00
```

> **Astuce :** tu peux aussi combiner création et permissions en une seule ligne :
> ```sh
> touch testShell00 && chmod 455 testShell00
> ```
